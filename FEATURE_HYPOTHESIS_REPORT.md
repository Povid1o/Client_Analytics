# Проверка гипотез feature engineering

## Методика

- Одна и та же модель: weighted-RMSE CatBoost на `target ** 0.25`, 1200
  итераций, одинаковые seed и постобработка по `salary_6to12m_avg`.
- Первичный screening всех групп выполнен на fold 0.
- Кандидаты проверены на дополнительных фолдах и на temporal holdout:
  январь–май → июнь.
- Региональный target statistic рассчитывался nested OOF: target validation
  никогда не участвовал в признаках train.
- Отрицательная delta означает улучшение WMAE.

## Первичный screening, fold 0

| Группа | WMAE | Δ к baseline | Вывод |
|---|---:|---:|---|
| baseline | 62 842 | — | — |
| согласие income anchors | 62 542 | −300 | перспективно на random CV |
| debt/income + utilization | 62 785 | −57 | дополнительного выигрыша нет |
| credit/debit flow balance | 62 556 | −286 | перспективно на random CV |
| тренды 3/6/12m | 62 624 | −218 | слабый плюс |
| региональная нормировка дохода | 62 716 | −126 | хуже anchors без нормировки |
| доли категорий расходов | 62 924 | +82 | ухудшение |
| log1p + percentile rank | 62 586 | −256 | лучший устойчивый кандидат |
| missing-флаги | 62 849 | +7 | полностью нейтрально |
| Bayesian region target encoding | 62 768 | −74 | эффект практически отсутствует |

`scale` и `regional_norm` используют агрегированный income anchor как
знаменатель, поэтому их честный инкремент следует сравнивать также с
`anchors` (62 542), а не только с baseline. По этому сравнению обе группы
ухудшают результат.

## Проверка устойчивости

### Flow balance

| Split | baseline | flows | Δ |
|---|---:|---:|---:|
| fold 0 | 62 842 | 62 556 | −286 |
| fold 1 | 61 901 | 61 672 | −229 |
| fold 2 | 62 638 | 62 610 | −28 |
| fold 3 | 62 327 | 62 412 | +85 |
| fold 4 | 62 171 | 62 475 | +304 |
| среднее | 62 376 | 62 345 | **−31** |
| temporal holdout | 60 637 | 60 638 | **+1** |

Итог: положительный эффект первых фолдов не воспроизводится. Добавлять эту
группу в финальный pipeline не следует.

### Log1p + percentile rank

| Split | baseline | log/rank | Δ |
|---|---:|---:|---:|
| fold 0 | 62 842 | 62 586 | −256 |
| fold 1 | 61 901 | 61 617 | −284 |
| fold 2 | 62 638 | 62 609 | −29 |
| fold 3 | 62 327 | 62 393 | +66 |
| fold 4 | 62 171 | 62 212 | +41 |
| среднее | 62 376 | 62 283 | **−92** |
| temporal holdout | 60 637 | 60 566 | **−71** |

Итог: единственная группа с положительным средним эффектом и на random CV,
и на temporal holdout. Однако выигрыш всего 0,1–0,2% и существенно меньше
статистической неопределённости CV, поэтому это экспериментальный кандидат,
а не доказанный прирост.

### Комбинация anchors + flows

На folds 0/1/2 delta составила −536/−606/−66, но temporal delta стала
**+518**. Отдельно anchors на temporal holdout дали **+261**. Это указывает
на временной drift качества/доступности источников дохода; комбинацию нельзя
переносить в test без time-aware калибровки.

Тренды на temporal holdout дали +62, то есть первичный выигрыш также не
подтвердился во времени.

## Практический вывод

CatBoost уже умеет самостоятельно строить разбиения по исходным суммам,
пропускам и категориям. Поэтому большинство ручных отношений лишь дублируют
доступную информацию и увеличивают пространство признаков. На этом датасете
основной прирост по-прежнему приходит от правильной постановки target/loss и
ансамблирования, а не от массового feature engineering.

Воспроизвести screening:

```bash
python3 evaluate_feature_hypotheses.py
```

Проверить конкретную комбинацию вместе с baseline:

```bash
python3 evaluate_feature_hypotheses.py --groups anchors,flows --compare
python3 evaluate_feature_hypotheses.py --groups log_rank --compare --time-holdout
```

Обучить отдельный экспериментальный submission с единственным устойчивым
кандидатом, не меняя основной `submission_improved.csv`:

```bash
python3 train_improved.py \
  --feature-groups log_rank \
  --output outputs/partial/submission_log_rank.csv
```

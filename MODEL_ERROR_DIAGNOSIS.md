# Почему модель ошибается и проверка residual stacking

## Где образуется WMAE

Диагностика выполнена на строгом outer fold. Base-прогноз для validation
обучен только на outer-train; обучающие прогнозы для residual-модели получены
через отдельный inner 4-fold CV.

- Base WMAE: **62 792**.
- Верхний target-дециль создаёт **45 821 WMAE**, то есть 73% всей ошибки.
- Верхний predicted-дециль создаёт **31 180 WMAE**, то есть половину ошибки.
- Но weighted-median residual внутри верхнего predicted-дециля равен всего
  **+319**. Следовательно, этот сегмент уже откалиброван: большие
  недооценки компенсируются большими переоценками похожих клиентов.
- Oracle global scale/offset улучшают WMAE лишь на 41–49.
- Даже oracle-коррекция по predicted-децилям улучшает только на 798.

Чтобы снизить общий WMAE ещё на 10 000, требуется уменьшить ошибку верхнего
predicted-дециля примерно на треть. Постобработка прогноза этого сделать не
может: нужен новый сигнал, разделяющий похожих по текущим данным клиентов с
доходом, например, 200 и 500 тысяч.

## Дефицит информации

| Сегмент | Доля общего веса | Локальный WMAE |
|---|---:|---:|
| `salary_6to12m_avg` отсутствует | 79,3% | 70 051 |
| `salary_6to12m_avg` присутствует | 20,7% | 34 950 |
| нет ни одного основного income anchor | 14,6% | 72 021 |
| есть ровно один anchor | 49,5% | 71 643 |
| есть три anchors | 19,4% | 46 075 |
| `first_salary_income` отсутствует | 87,8% | 66 884 |
| `first_salary_income` присутствует | 12,2% | 33 383 |

Ошибка резко падает, когда доступны несколько качественных независимых
источников дохода. На большинстве строк модель получает один шумный proxy и
не может восстановить индивидуальное отклонение target от него.

## Residual CatBoost

Проверены compact и full residual-модели с correction shrinkage 0.25–1.0.

| Residual model | Shrinkage | Δ WMAE |
|---|---:|---:|
| full features | 0.25 | +328 |
| compact features | 0.25 | +525 |
| full features | 1.00 | +3 132 |
| compact features | 1.00 | +3 850 |

Даже сильный shrinkage ухудшает результат. Residual learner повторно
использует те же признаки, но не получает новой информации; он учится на
нестабильных ошибках отдельных inner-моделей и переносит этот шум на outer
validation.

Отдельные модели по полу также ухудшили WMAE на 739. Большая ошибка мужского
сегмента вызвана более тяжёлым target-хвостом, а не недостатком ёмкости общей
модели.

## Эксперименты с наибольшим оставшимся потенциалом

1. **Tail classifier + specialists.** OOF-классификаторы порогов 150k/300k/
   500k, отдельные регрессоры для tail и non-tail, мягкий gate по вероятности.
   Для `target >= 300k` текущий порог `prediction >= 250k` даёт weighted
   precision/recall около 0,71/0,71 — здесь ещё есть пространство для роста.
2. **Другая модельная семья.** LightGBM/XGBoost/TabM или нейросеть, затем
   оптимизация OOF-blend. Ещё один CatBoost на тех же признаках слишком
   коррелирован с base; прирост возможен только через разнообразие ошибок.
3. **Distributional/quantile ensemble.** Несколько conditional quantiles и
   отдельная tail probability вместо одного point estimate. Gate и blend
   обязательно подбираются только по OOF.
4. **Source-specific experts.** Отдельные модели для наблюдаемых режимов
   наличия salary/ILS/payout anchors, но с мягким blending, чтобы не терять
   объём train. Жёсткое разбиение выборки, как эксперимент по полу, опасно.
5. **Leaf-space neighbours.** KNN/weighted median по CatBoost leaf embeddings
   как действительно иной локальный estimator; проверять только OOF.

Цель −10 000 без новых данных крайне агрессивна. Она требует не калибровки,
а существенно более точного распознавания клиентов верхнего хвоста.
